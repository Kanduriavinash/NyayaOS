# NyayaOS: The Agentic Operating System for the Justice Ecosystem

⚖️ **Hackathon Track:** Open Innovation (Legal Technology)  
🚀 **Tech Stack:** Next.js (React/TypeScript), Tailwind CSS, Prisma, SQLite, and **NitroStack** (Official MCP Framework).

---

## ⚖️ Overview & Vision

**NyayaOS** is a decentralized, agentic AI platform built on the Model Context Protocol (MCP) designed to streamline and automate the entire lifecycle of judicial cases. Rather than automating a single isolated legal task, NyayaOS connects citizens, lawyers, court registries, and judges through specialized MCP servers, coordinating files, schedules, and analytics dynamically.

Its purpose is **not** to replace human legal experts or judges. Instead, it addresses system-wide judicial administrative delays, filing errors, unnecessary court visits, and fragmented communications by orchestrating every stage of a case.

---

## 🚀 Core Features & Portals

NyayaOS provides 5 dedicated web portals to serve all primary stakeholders:

### 1. 🧑‍💼 Citizen Portal (Port 3000)
* **Intake Interface:** Citizens can describe their legal grievances in natural language. An intake agent classifies the issue (e.g., property dispute, family law).
* **Document Uploader:** Structured file uploads for proof of identity and evidence.
* **Intelligent Feedback:** Real-time analysis of uploaded documents to notify users of missing pages, signatures, or formats.
* **Case Tracking:** A user-friendly dashboard to track filing status, scheduled hearings, and issued court orders.

### 2. ⚖️ Lawyer Workspace (Port 3001)
* **Case Claiming:** Review and claim cases requiring legal representation.
* **AI Drafts:** Automated draft generation for plaints and petitions based on client intake.
* **Precedent Finder:** Search tool that displays relevant case-law, statutes, and legal precedents with citations.
* **Hearing Tracker:** Calendar interface to manage client representations and upcoming hearing dates.

### 3. 👨‍⚖️ Judge Chambers (Port 3002)
* **Smart Case Briefs:** Concise case summaries, timelines, and index of uploaded evidence generated by the Judge Assistant Agent.
* **Order Issuer:** Interface to create, sign, and issue formal court orders and directions.
* **Calendar Management:** Clear view of daily and weekly courtroom schedules.
* **No Verdict Suggestion:** Built strictly to assist—not replace—judicial decision-makers.

### 4. 🗄️ Registry Dashboard (Port 3003)
* **Filing Audit:** Automatic checklist matching to ensure all required filings are complete.
* **Jurisdiction Validation:** Validates that the case has been filed in the correct geographical and subject-matter jurisdiction.
* **Manual & Auto Approvals:** Allows registry officers to review automated checks and finalize the case docket registry.

### 5. ⚙️ Central Admin Console (Port 3004)
* **Global Metrics:** Real-time view of system user counts, active cases, and audit logs.
* **MCP Server Monitor:** Status logs for the connected MCP servers.
* **System Settings:** Control configurations for the multi-agent orchestration.

## ⚡ Multi-Agent & MCP Architecture

At the core of NyayaOS is an orchestration layer powered by the **Model Context Protocol (MCP)**. Our 8 MCP servers are bootstrapped using **NitroStack** decorators (such as `@McpApp`, `@Module`, `@Tool`, `@Resource`, `@Prompt`) running in dual transport mode (supporting both STDIO and HTTP Server-Sent Events).

### The 8 NitroStack MCP Servers (Ports 4001 - 4008)

1. **Identity MCP (`mcp/identity-server` - Port 4001)**
   * *Tools:* Identity verification, OTP authentication.
   * *Resources:* System identity verification rules.
2. **Document MCP (`mcp/document-server` - Port 4002)**
   * *Tools:* OCR text extraction, signature detection, page completeness checks.
   * *Resources:* Required document checklists, standard filing templates.
3. **Registry MCP (`mcp/registry-server` - Port 4003)**
   * *Tools:* Jurisdiction check, court fee calculator, filing schema validation.
4. **Research MCP (`mcp/research-server` - Port 4004)**
   * *Tools:* Search statutes, fetch judgments, find relevant case precedents.
   * *Resources:* Statutory acts, court rules, public case-law database.
5. **Scheduling MCP (`mcp/scheduling-server` - Port 4005)**
   * *Tools:* Courtroom availability check, hearing allocation, judge calendar syncing.
6. **Notification MCP (`mcp/notification-server` - Port 4006)**
   * *Tools:* Automated email, SMS alerts, and browser push notifications.
7. **Analytics MCP (`mcp/analytics-server` - Port 4007)**
   * *Tools:* Fetch case load, average disposal time, registry rejection rates, and judge workload.
8. **Knowledge MCP (`mcp/knowledge-server` - Port 4008)**
   * *Tools:* Browse legal guides, acts indexes, and courtroom protocols.

## 🔄 High-Level Case Lifecycle Workflow

Below is the automated sequence of events when a citizen submits a dispute:

```
[Citizen Intake] ──> [Planner Agent] ──> [Identity MCP] (Verify User)
                                                │
[Registry MCP] <── [Document MCP] (OCR & Sign) <┘
      │
      ├──> [Registry Approves & Generates Docket]
      │
      └──> [Research MCP] (Precedents) ──> [Scheduling MCP] (Book Hearing)
                                                    │
[Judge Dashboard] <── [Notification MCP] (Alerts) <─┘
```

## 🛠️ Getting Started & Installation

### 1. Prerequisites
* **Node.js** (v18 or higher)
* **pnpm** package manager (v10+)

### 2. Install Dependencies
Run the installation command in the workspace root:
```bash
pnpm install
```

### 3. Initialize SQLite Database & Seed Data
Generate the Prisma Client, push the database schema, and seed default records (lawyers, judges, courts, cases):
```bash
# Push schema to local SQLite database file
pnpm --filter @nyayaos/database prisma db push

# Seed the database with mock records
pnpm --filter @nyayaos/database db:seed
```

### 4. Build the Monorepo
Compile all TypeScript packages, MCP servers, and Next.js applications:
```bash
pnpm build
```

---

## 🖥️ Running the Applications

### Start the 8 MCP Servers (Dual STDIO + HTTP SSE Transport)
Run the following command to spin up all servers on ports `4001` through `4008`:
```bash
pnpm --filter "@nyayaos/*-server" start:prod
```

### Start the Next.js Dashboards
Run the portals in development mode concurrently:
```bash
pnpm dev
```

Open the following URLs in your browser:
* **Citizen Portal:** [http://localhost:3000](http://localhost:3000)
* **Lawyer Workspace:** [http://localhost:3001](http://localhost:3001)
* **Judge Chambers:** [http://localhost:3002](http://localhost:3002)
* **Registry Dashboard:** [http://localhost:3003](http://localhost:3003)
* **Central Admin Console:** [http://localhost:3004](http://localhost:3004)

## 🔌 Connecting to NitroStudio

You can load and test all tools, resources, and prompts inside **NitroStudio**:

### Option A: STDIO Connection (Recommended)
1. Open **NitroStudio**.
2. Click **Add Server** → Select the **Nitro Project** tab.
3. Browse to the folder of the server you want to connect (e.g. `mcp/identity-server` or `mcp/document-server`).
4. Click **Open Project** → Select **Studio App Canvas**.

### Option B: HTTP Stream Connection
Since the servers run in dual transport mode in production, you can link them directly via URL:
1. Open **NitroStudio**.
2. Click **Add Server** → Select the **Other Project** tab.
3. Name your project (e.g. `document-server`).
4. Set Connection Type to **HTTP (Streamable HTTP)**.
5. Set URL to: `http://localhost:4002/mcp` (substitute the port `4001`-`4008` for other servers).
6. Click **Add Server**.

## ☁️ Deploying to NitroCloud

To deploy your server to the official NitroCloud hosting platform:
1. Log into your NitroStack CLI:
   ```bash
   npx @nitrostack/cli login
   ```
2. Navigate to the specific server folder:
   ```bash
   cd mcp/identity-server
   ```
3. Build and deploy:
   ```bash
   npx @nitrostack/cli build
   npx @nitrostack/cli deploy
   ```
   *(Or link it inside NitroStudio and click the **Deploy** button in the visual interface)*
